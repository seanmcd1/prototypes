Notion Docs API

Programmatic CRUD for Notion documents — blocks, rich text, versioning, and more.

All requests require an Authorization header with a valid bearer token.

Authentication

Secure, scoped access to workspaces and documents.

🔑

All API requests must include an Authorization: Bearer <token> header and the Notion-Version header.

Auth Methods

MethodUse CaseToken Prefix
OAuth 2.0Public integrations & partner appsntn_oauth_*
Internal TokenPrivate workspace automationsntn_*

OAuth 2.0 Flow

Step 1
Redirect
Send user to Notion's auth URL with scopes
→
Step 2
Authorize
User selects workspace and grants access
→
Step 3
Exchange
Swap auth code for access + refresh tokens
→
Step 4
Call API
Use Bearer token on all subsequent calls

Scopes

ScopeAccessDescription
docs:readReadRead document content, metadata, and blocks
docs:writeWriteCreate, update, and archive documents
workspace:readReadList workspaces and permissions
comments:readReadRead document comments Beta
comments:writeWriteCreate and resolve comments Beta
⚠️

Default deny: Tokens have no access until scopes are explicitly granted. Always request the minimum scopes needed.

Token Exchange

Request
POST https://api.notion.com/v1/oauth/token { "grant_type": "authorization_code", "code": "abc123...", "redirect_uri": "https://yourapp.com/callback" } // Response { "access_token": "ntn_oauth_v2_abc...", "token_type": "bearer", "expires_in": 3600, "refresh_token": "ntn_refresh_xyz...", "workspace_id": "ws_abc123", "scopes": ["docs:read", "docs:write"] }

Content Model

Documents are structured trees of typed blocks, each with a stable ID.

Document Structure

Every document is a tree where the root represents the page, and children are content blocks. Each block has a globally unique ID that persists across edits.

📄 document doc_8f3a2b1c
├─ heading_1 blk_001 — "Project Overview"
├─ paragraph blk_002 — "This document outlines…"
├─ heading_2 blk_003 — "Requirements"
├─ bulleted_list blk_004
├─ list_item blk_005 — "Auth support"
├─ list_item blk_006 — "CRUD operations"
└─ list_item blk_007 — "Versioning"
├─ divider blk_008
└─ callout blk_009 — "⚡ See API playground"

Update Document

Update document metadata or modify existing blocks with conflict detection.

PATCH/v1/docs/{doc_id}🔐 docs:write
⚠️

Optimistic locking required. Include If-Match header with the document's current ETag. Stale ETags return 409 Conflict.

Required Headers

HeaderDescription
If-MatchETag from the last GET response
Idempotency-KeyUnique key for safe retries

Request Body

FieldTypeDescription
titlestringNew document title
iconstringNew icon emoji or URL
blocksarrayBlocks to update (by block ID)

Example — Update a block

Request
PATCH /v1/docs/doc_8f3a2b1c If-Match: "W/\"v3-def456\"" Idempotency-Key: "idem_update_001" { "blocks": [ { "id": "blk_002", "content": { "rich_text": [ { "text": "Updated paragraph content." } ] } } ] }

Conflict Response — 409 Conflict

Error
{ "object": "error", "code": "conflict", "message": "Document has been modified since your last read.", "current_version": 5, "your_version": 3, "current_etag": "W/\"v5-ghi789\"", "request_id": "req_e5f6a7b8" }

Archive Document

Soft-delete a document by moving it to the archive. Recoverable for 30 days.

DELETE/v1/docs/{doc_id}🔐 docs:write

Query Parameters

ParamTypeDefaultDescription
permanentbooleanfalsePermanently delete (admin only)

Response — 200 OK

Response
{ "id": "doc_8f3a2b1c", "object": "document", "archived": true, "archived_time": "2024-06-08T16:00:00Z", "recoverable_until": "2024-07-08T16:00:00Z", "request_id": "req_d3e4f5a6" }
🗑️

Archived documents are recoverable for 30 days. After that, permanently deleted.

Versioning & Conflict Detection

Every write creates a new version. Use ETags to prevent lost updates.

How It Works

Documents use an incrementing version counter and ETags for optimistic concurrency control.

1
GET Document
Response includes ETag header
→
2
PATCH with If-Match
Send ETag in If-Match header
→
3
Success or Conflict
200 OK or 409 with current version

Version History

Conflict Resolution Pattern

Recommended
// 1. Fetch current document const doc = await fetch(`/v1/docs/${docId}`); const etag = doc.headers.get("ETag"); // 2. Attempt update with ETag const res = await fetch(`/v1/docs/${docId}`, { method: "PATCH", headers: { "If-Match": etag }, body: JSON.stringify({ /* changes */ }) }); // 3. Handle conflict if (res.status === 409) { const conflict = await res.json(); // Re-fetch, merge changes, retry with new etag }

Handling 429 Responses

Retry Pattern
async function callWithRetry(url, options, maxRetries = 3) { for (let attempt = 0; attempt < maxRetries; attempt++) { const res = await fetch(url, options); if (res.status !== 429) return res; const retryAfter = parseInt(res.headers.get("Retry-After")) || 1; await new Promise(r => setTimeout(r, retryAfter * 1000)); } throw new Error("Max retries exceeded"); }
